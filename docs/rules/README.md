# Rules

The project's enforced rule set. Every change must follow every file in this directory.

Add a new file here for a new rule category; it is picked up automatically.

- `coding-style.md`: naming, formatting, file/source layout, language idioms, module style, frontend conventions, versioning.
- `testing.md`: test runners, layout, what must be tested, how to run the suites.
- `documentation.md`: required documents and doc-currency expectations.
- `security.md`: the read-only kubectl invariant, the audit log, and the local-only binding.
