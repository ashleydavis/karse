# guided-commands

## Overview

A **Commands** tab on each resource detail page (pod, node, deployment, stateful set, daemon set) that lists suggested kubectl commands for that resource. The commands are pure display strings the user copies and runs themselves. Nothing in this feature executes commands or wires them to any adapter, so it does not violate the read-only invariant (see `read-only-invariant`).

Backed by: `frontend/src/lib/guided-commands.ts`, `frontend/src/components/commands-tab.tsx`.

## Behaviour

- `buildGuidedCommands(target)` returns a list of `{ label, command }` for the target resource. It is a pure function: no side effects, no execution.
- Pod commands: describe, view logs, follow logs, open a shell, get YAML, delete. A namespace, when present, is appended as `-n <namespace>`.
- Node commands: describe, get YAML, list pods on node, cordon, drain (`--ignore-daemonsets --delete-emptydir-data`), uncordon.
- Workload commands (deployment/statefulset/daemonset): describe, get YAML, restart rollout, rollout status, and delete; deployments and stateful sets also get a scale-replicas suggestion.
- The Commands tab renders each suggestion as a labelled code block with a copy-to-clipboard button, and a prominent read-only note: "Karse is read-only and never runs these commands. Copy and run them yourself."
- Commands word-wrap so the full command is always visible without horizontal scrolling, and the tab uses the full detail-page width.
- A search box filters the command list (`filterGuidedCommands(commands, query)`), fuzzy-matching the query against each command's label and command text. An empty query shows the full list.
- Some suggestions are mutating kubectl commands (delete, scale, drain, cordon). They are text only; Karse never runs them.

## Acceptance Criteria

- [x] Command suggestions are generated by a pure function with no execution.
- [x] Pod, node, and workload (deployment/statefulset/daemonset) targets each have a tailored command list.
- [x] A namespace, when present, is appended as `-n <namespace>`.
- [x] The Commands tab renders each command with a copy button and a read-only disclaimer.
- [x] Commands word-wrap (no horizontal scroll) and the tab uses the full detail-page width.
- [x] The command list is searchable via a filter box.
- [x] Mutating suggestions are shown as copyable text only and are never executed by Karse.
