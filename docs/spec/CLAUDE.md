# Spec rules

- The spec is the source of truth. Never invent behaviour not in the spec.
- When editing `detail.md`, update the sibling `index.md` status fields if they no longer match.
- When a change lands that meets acceptance criteria, tick the matching boxes in the relevant `detail.md`. If all are ticked, set `**Implementation:** Complete`.
- Do not mark `**Spec:** Settled` while Open Questions remain unresolved.
- Mirror any structural change under `docs/testing-manual/`.
